Documentation for $.qr()

Defaults

The qr plugin does not require any parameters, it uses a set of defaults which should work for most text lengths.

For large amounts of text, an overflow may occur. 

$('#defaults').qr();
$('#defaults').qr with no options passed

Options

Each of the options that the plug-in takes are details below

typeNumber

The type of QR code can specified. This can be in the range 1-10 (default 4)

For larger amounts of data, you will need to increase this value to avoid overflow 

$('.type_test').each(function () {
    $(this).qr({typeNumber: $(this).data('type')});
});

errorCorrectLevel

The amount of error correct used in the QR code can specified. This can be one of 'L','M','Q','H'. (default 'M')

For higher degrees of error correct, you may need to increase the value of typeNumber to avoid overflow 

$('.error_correct_test').each(function () {
    $(this).qr({errorCorrectLevel: $(this).data('error-correct-level')});
});

cellSize

To alter the size of the QR code, use the cellSize parameter (default 2). This value is in pixels. 

$('.cell_size_test').each(function () {
    $(this).qr({cellSize: $(this).data('cell-size')});
});

marginSize

A margin can be specified within the QR code itself (default 2). This value is in pixels.

For most instances this can be dropped to 0 and CSS used instead. 

$('.margin_size_test').each(function () {
    $(this).qr({marginSize: $(this).data('margin-size')});
});

source

There are three options for dictating the source (i.e. input text) for the QR code.

  • The textual contents of the element (default)
  • Some specified text (i.e. a string)
  • A callable which returns a string

The contents of the element itself

$('#use-text').qr();
Text in this element

Specified text

$('#specified-text').qr({source: 'Something else'});
Don't use this!

Text from a callable

The function receives the element in question as this, so that further work can be done to access, e.g. attributes. 

$('#generated-text').qr({source: function () { return this.attr('title'); }});
Don't use this!

target

If you want to map the resulting QR code to one or more jQuery elements you can specify the target. By default this will be the element itself. 

$('#multi-target').qr({target: $('.target')});

This will be output on the targets

outputType

Three output types are supported by the plug-in. These are:

  • canvas — requires compatible browser
  • image — base64 encoded GIF, requires compatible browser
  • table — should work cross-browser

By default the plug-in will check the browser for compatibility with the various output formats working from the top of the above list downwards.

Unless you require a particular output format (such as an image that can be saved from the browser) it is best to let the plugin determine the capabilities of the browser. 

$('.outputTypes').each(function () {
    $(this).qr({outputType: $(this).data('output-type')});
});
